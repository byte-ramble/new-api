## 3. 我们如何共享

### 3.1 上游 LLM 服务商 ⚠️ 重要

**这是您必须知道的最重要的事**：

当您调用 OmniRouter 时，您的 prompt 会被**实时转发到您选择的分组对应的上游 LLM 服务商**（如 OpenAI、Anthropic、AWS Bedrock 等）。这些上游服务商：
- 会处理您的 prompt 来生成回复
- 会按其各自的隐私政策保留和使用数据
- 部分可能将您的输入用于改进模型（除非您选择了 zero-retention 分组）
- 部分会把数据存在中国大陆境外（影响数据跨境）

**主要上游服务商隐私政策链接**（您应当自行阅读）：
- OpenAI: https://openai.com/policies/privacy-policy
- Anthropic: https://www.anthropic.com/legal/privacy
- Google: https://policies.google.com/privacy
- AWS Bedrock: https://aws.amazon.com/bedrock/data-protection/

我们尽量提供 zero-retention / 不训练 的上游分组（如 Anthropic 默认不训练、AWS Bedrock 默认不储存）。具体每个分组的数据政策见 [{{DOMAIN}}/pricing]({{DOMAIN}}/pricing) 分组介绍。

### 3.2 服务提供商
| 类型 | 服务商 | 共享数据 | 用途 |
|---|---|---|---|
| 支付 | Stripe / Creem / 支付宝 / 微信 | 邮箱、订单金额 | 完成支付 |
| 邮件 | SendGrid / 阿里云邮件推送（按实际配置） | 邮箱、邮件内容 | 系统邮件 |
| 监控 | Prometheus / Grafana（自部署） | 内部使用 | 服务监控 |
| 托管 | DigitalOcean / 阿里云 / 腾讯云（按实际配置） | 全部应用数据 | 基础设施 |

我们与上述服务商有数据处理协议（DPA），要求他们仅按我们的指示处理数据。

### 3.3 法律要求
当我们收到 **合法的** 政府 / 司法机关的请求时，可能配合披露数据。我们会评估请求合法性、最小化披露范围、并尽量在法律允许时通知您。

### 3.4 业务转让
如发生合并、收购、资产转让，您的数据可能转移给受让方。我们会通知您并给您选择删除数据的机会。

### 3.5 我们 **不会做** 的
- ❌ 出售个人数据给第三方
- ❌ 把您的 prompt 数据用于训练我们的模型
- ❌ 跟广告平台共享个人数据

## 4. 数据存储

### 4.1 存储位置
- 主数据库 / Redis：**{{JURISDICTION}}** 境内的数据中心（按实际部署）
- CDN / Cloudflare：全球边缘节点（仅缓存静态资源，不缓存请求内容）
- 上游 LLM 调用：转发到对应服务商所在地区（多在美国 / 新加坡 / 日本）

### 4.2 存储期限
| 数据 | 期限 |
|---|---|
| 账户信息（活跃账户） | 直至您注销 |
| 账户信息（注销后） | 30 天后删除 / 匿名化 |
| 计费日志 | **6 个月**（满足税务 / 审计） |
| 操作审计日志 | **180 天** |
| 充值订单 | **3 年**（财税合规） |
| 错误日志 | **30 天** |
| **prompt / response 内容** | **不存储** |

### 4.3 加密
- 传输：全程 HTTPS / TLS 1.2+
- 静态：账户密码 bcrypt 哈希；API Key 哈希存储；数据库磁盘加密
- 敏感操作（充值、退款、API Key 创建）需要二次确认

## 8. 跨境数据转移

由于上游 LLM 主要在境外（美国 / 新加坡），您的 prompt 在转发过程中会跨境传输。我们采取的合规措施：
- **告知透明**：在分组介绍中明确标注上游所在地
- **协议保障**：与跨境服务商签订标准合同条款

- **零持久化**：上游处理完后我们不留副本

如您不接受跨境数据传输，请使用我们的 **国产模型分组**（DeepSeek / 通义 / 智谱 / Kimi / 文心等，全部境内闭环）。
